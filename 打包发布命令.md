# dwsal CLI 打包发布操作手册

## ⚙️ 动态参数（每次发布前修改这里）

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `VERSION` | `1.0.8` | 新版本号，同时用于 npm 和 GitHub Release |
| `OLD_VERSION` | `1.0.7` | 上一个需要废弃的旧版本号 |
| `RELEASE_NOTES` | `更新说明` | GitHub Release 的版本说明文字 |

> 修改完上表后，将下方所有命令中对应占位符替换再执行。

---

## 步骤一：修改 npm 版本号

编辑文件：`Jusoft.DingTalk.MCP.CLI\npm-wrapper\package.json`

将 `"version"` 字段改为新版本号（与上表 `VERSION` 一致）：

```json
"version": "1.0.8"
```

---

## 步骤二：多平台编译打包

```powershell
cd "e:\工作文件\鸿欢团队资源管理项目\2025年\云效代码\product-public-module\Jusoft.DingTalk.MCP.CLI"

# Windows x64
dotnet publish -r win-x64 -c Release --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o "publish\win-x64"

# Linux x64
dotnet publish -r linux-x64 -c Release --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o "publish\linux-x64"

# macOS arm64（Apple Silicon）
dotnet publish -r osx-arm64 -c Release --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o "publish\osx-arm64"
```

---

## 步骤三：推送代码到 GitHub（云效同步）

```powershell
cd "e:\工作文件\鸿欢团队资源管理项目\2025年\云效代码\product-public-module"
git push origin master
```

---

## 步骤四：发布到 npm

```powershell
cd "e:\工作文件\鸿欢团队资源管理项目\2025年\云效代码\product-public-module\Jusoft.DingTalk.MCP.CLI\npm-wrapper"
npm publish --access public
```

---

## 步骤五：创建 GitHub Release 并上传 zip

> 将 `v1.0.8` 和 `更新说明` 替换为上表中的 `VERSION` 和 `RELEASE_NOTES`。

```powershell
gh release create v1.0.8 `
  "e:\工作文件\鸿欢团队资源管理项目\2025年\云效代码\product-public-module\Jusoft.DingTalk.MCP.CLI\publish\win-x64\dwsal-win-x64.zip" `
  --repo QiChangWei/-CLI `
  --title "v1.0.8" `
  --notes "更新说明"
```

---

## 步骤六：废弃旧版 npm 包

> 将 `1.0.7` 替换为上表中的 `OLD_VERSION`。

```powershell
npm deprecate dwsal@1.0.7 "已废弃，请升级至最新版本: npm install -g dwsal@latest"
```

> npm 规定：包发布超过 72 小时后不可删除，`deprecate` 是唯一的官方做法，用户安装旧版时会收到警告提示。

---

## 附：删除 GitHub Release / Tag（回滚用）

```powershell
# 仅删除 release，保留 tag
gh release delete v1.0.8 --repo QiChangWei/-CLI --yes

# 同时删除对应的 git tag
gh release delete v1.0.8 --repo QiChangWei/-CLI --yes --delete-tag
```

> 参数说明：`--yes` 跳过确认，`--delete-tag` 同时删除 git tag。
